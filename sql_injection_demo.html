WHAT IS SQL INJECTION?

🎯

DEFINITION

SQL Injection (SQLi) is an attack technique where malicious SQL code is inserted into an input field, causing the backend database to execute unintended queries — exposing, modifying, or deleting data.

⚙️

HOW IT HAPPENS

When user input is concatenated directly into a SQL query without sanitization, an attacker can close the original query with ' and append their own SQL commands.

💥

IMPACT

Authentication bypass, unauthorized data access, data tampering, full database dump, and in some cases, remote code execution on the server.

📊

PREVALENCE

SQLi consistently ranks in the OWASP Top 10 Web Vulnerabilities. It has been used in major breaches at Sony, Yahoo, LinkedIn, and thousands of smaller websites globally.

04

PREVENTION TECHNIQUES

PREPARED STATEMENTS

Use parameterized queries where user input is passed separately — never concatenated into SQL.

$stmt = $pdo->prepare( "SELECT * FROM products WHERE name = ?" ); $stmt->execute([$input]);

STORED PROCEDURES

Encapsulate SQL logic in database-side procedures, preventing direct query manipulation from application code.

CALL sp_GetProduct( @ProductName );

INPUT VALIDATION

Whitelist valid characters, reject unexpected symbols like ' -- ; /*. Never rely on blacklisting alone.

// Allow only alphanumeric if(!preg_match('/^[a-zA-Z0-9 ]+$/', $input)) die("Invalid!");

LEAST PRIVILEGE

Database accounts used by the app should only have SELECT/INSERT on required tables — never admin or DROP privileges.

GRANT SELECT, INSERT ON shopzone.products TO 'app_user'@'localhost';

ORM / QUERY BUILDERS

Use libraries like Hibernate, SQLAlchemy, or Eloquent. They auto-escape inputs and rarely expose raw SQL to user data.

// Laravel / Eloquent Product::where('name', $input)->get();

WEB APPLICATION FIREWALL

Deploy a WAF (e.g., ModSecurity, Cloudflare WAF) to detect and block SQLi patterns before they reach the application.

SecRule ARGS "@detectSQLi" "id:1001,phase:2, deny,msg:'SQLi'"
